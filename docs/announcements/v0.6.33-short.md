# Vault-for-LLM v0.6.33

This release makes advanced Supabase sharing more concrete for multi-agent setups.

What's new:

- Added `docs/supabase_read_policy.sql`, a ready-to-paste Supabase read-policy template.
- `vault setup-agent --supabase-setup advanced` now writes `agent-install/supabase-read-policy.sql`.
- The template creates a read-only `vault_search_readable` RPC for hosted agents, Coze, n8n, and cross-host readers.
- The RPC applies `scope`, `sensitivity`, `owner_agent`, `allowed_agents`, and `expires_at` filters.
- It returns safe metadata and summaries only; raw full text stays out of the default hosted read path.

Core Vault remains local-first. Supabase is still optional and should be used only when you need cross-host, hosted-agent, or team memory sharing.

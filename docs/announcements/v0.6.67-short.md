# Vault-for-LLM v0.6.67

This release adds the MCP path for session capture.

## What's New

- Added `vault_capture_session` to the `review`, `maintenance`, and `full` MCP
  tool profiles.
- The tool runs the same extractor as `vault capture session`.
- It previews by default and requires `write_candidates=true` before anything is
  written to `memory_candidates`.
- It stays out of the `core` profile so everyday agent sessions keep a smaller
  tool schema.
- Relative transcript paths resolve under the current project directory.
  Absolute paths require `allow_absolute_path=true`.

## Why It Matters

The memory loop now has an MCP-native ingestion path:

```text
agent session -> vault_capture_session preview -> candidate queue -> review inbox -> promotion or rejection feedback
```

That means an agent can turn a finished work session into reviewable lessons
without needing a separate CLI handoff, while the safety boundary remains the
same: candidate-first, privacy-gated, no auto-promotion.

## Try It

```bash
pip install "vault-for-llm[mcp]==0.6.67"
vault-mcp --project-dir ~/Vaults/my-project --tool-profile review
```

Then call `vault_capture_session` with a transcript path. Start with
`write_candidates=false`; only set `write_candidates=true` after reviewing the
preview.

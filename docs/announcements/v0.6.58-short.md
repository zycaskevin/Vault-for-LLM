# Vault-for-LLM v0.6.58

This release adds `vault automation cycle`, a one-command safe learning loop for memory maintenance.

The cycle reads reviewed candidate outcomes, writes a bounded `learning_policy.json`, then runs policy-based automation so Dream can use the latest curation hints. It helps agents sort better cleanup candidates first while keeping the important boundary intact: no auto-promotion, no hard delete, and no privacy or access-policy bypass.

Why it matters: Vault memory automation can now learn from review feedback without turning into an uncontrolled self-writing memory system.

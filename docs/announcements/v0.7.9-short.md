# Vault-for-LLM v0.7.9

This patch makes automatic memory ingestion easier to observe.

`vault memory pipeline` now supports `--write-report`, which writes:

- `reports/automation/pipeline-latest.json`
- `reports/automation/pipeline-latest.md`

The report is a compact ingestion receipt: discovered transcript count,
processed capture count, candidate write/preview/reject counts, gate status, and
next action. It does not store raw candidate body fields, content previews, or
gate payloads in the persisted receipt.

Generated setup-agent cron, LaunchAgent, and n8n schedules now include
`--write-report` on the memory pipeline step, so scheduled memory ingestion
leaves a visible handoff artifact by default.

Safety stays candidate-first: this does not promote active memory, read extra
transcripts beyond the selected pipeline inputs, or change privacy gates.

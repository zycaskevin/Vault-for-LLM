# Issue 發送技能

## 描述
自動化識別問題歸屬並發送 GitHub Issue 的技能。

## 使用場景
- 發現開源插件有性能問題或 bug
- 需要向開源項目提交優化建議
- 需要發送高質量的 Issue（包含測試數據）

## 核心功能

### 1. 問題歸屬識別
自動判斷問題是原作者的問題還是我們自己改動造成的：
- 檢查配置文件確認我們是否修改過
- 對比原版設計和我們的配置
- 如果是我們自己改的，不發 Issue

### 2. Issue 生成
生成標準格式的 Issue：
- 清晰的標題
- 詳細的問題描述
- 測試數據支持
- 解決方案建議
- 風險評估

### 3. 自動發送
使用 GitHub API 發送 Issue：
- 需要 GitHub Token（存儲在 .env.secrets）
- 處理權限問題（如無權限創建 labels）

## 使用方法

### 準備工作
1. 確認 GitHub Token 已配置
2. 確認問題歸屬（原作者 vs 我們自己改動）
3. 準備測試數據

### 發送命令
```
使用此技能發送 GitHub Issue：
1. 確認問題歸屬
2. 準備 Issue 內容
3. 發送到目標倉庫
```

## 技能文件
- `SKILL.md` - 技能說明文檔
- `send-issue.js` - 發送腳本

## 範例

### 成功案例
- A2A Gateway Issue #32（消息壓縮 60%，延遲降低 50-60%）
- 提供了完整的測試數據和解決方案

### 避免的坑
- 不要只報問題，要提供解決方案
- 要有實際測試數據支持
- 避免硬編碼
- 發 Issue 前先確認問題歸屬

## 環境變量
- `GITHUB_TOKEN` - GitHub 個人訪問令牌

## 依賴
- requests 庫
- GitHub API

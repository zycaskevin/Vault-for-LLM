# Vault-for-LLM v0.6.98

This release makes memory automation easier to trust.

`vault automation brief` now shows an explainable `importance_score` for
top-used memories. Instead of only counting access, the brief explains the
signals behind the ranking: citations, recency, trust, freshness, TTL pressure,
and governance protection hints.

Why it matters:

- agents can see which memories deserve protection before forgetting
- expired-but-used memories are easier to refresh or cold-store
- humans get a shorter, clearer 5% review surface
- the score remains guidance only, not an override for privacy, access policy,
  promotion, archive, or deletion

Upgrade:

```bash
pip install -U "vault-for-llm[mcp]==0.6.98"
vault automation brief --pretty
```

# Supabase UPSERT 重複

Supabase L2 UPSERT 重複插入問題，因 content_hash 包含時間戳導致 hash 不一致，且舊 title 記錄與新記錄衝突。

## 關鍵要點
- content_hash 使用包含時間戳的內容計算，導致每次執行 hash 不同
- 舊 title 記錄（如 'active'、'current'）與新記錄衝突
- 修復包括改用 stable_hash、刪除重複記錄與清理遺留資料
- 24 筆資料減少至 3 筆

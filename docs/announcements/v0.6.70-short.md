# Vault-for-LLM v0.6.70

This release lets scheduled memory automation handoffs include uncaptured
transcript hints when the installer explicitly opts in.

## What Changed

- Added `vault setup-agent --automation-include-transcripts`.
- Added `--automation-transcript-limit` for generated automation schedules.
- Cron, LaunchAgent, and n8n templates can now run:

```bash
vault automation cycle ... &&
vault automation inbox --write-handoff --include-transcripts
```

The transcript discovery remains metadata-only. It lists likely transcript
paths for the next reviewer, but does not read transcript contents, write memory
candidates, or promote active knowledge.

## Why It Matters

The closed loop now has a cleaner daily handoff:

```text
scheduled automation -> inbox-latest.json -> review queue + uncaptured transcript paths
```

That helps the next agent decide what should be captured without dumping raw
conversation exports into context.

## Try It

```bash
pip install "vault-for-llm[mcp]==0.6.70"

vault setup-agent \
  --non-interactive \
  --agent automation-agent \
  --scope shared \
  --agent-project-dir ~/Vaults/my-project \
  --features core,mcp,memory_agents \
  --automation-schedule cron \
  --automation-include-transcripts \
  --json
```

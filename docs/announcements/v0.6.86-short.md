# Vault-for-LLM v0.6.86

This release completes the MCP side of the multi-Agent update loop.

`vault_update_status` now supports:

```json
{
  "doctor": true,
  "agent_id": "codex",
  "max_status_age_minutes": 1440
}
```

That means MCP-only agents can check the same health state as:

```bash
vault agent doctor
vault update-status --doctor
```

It reports:

- whether the shared `update-status.json` exists
- whether the notice is fresh
- whether the notice was written by the current Vault runtime
- whether every registered Agent appears in `agent_update_notices`
- which local runtimes need restart, re-register, or upgrade

`setup-agent` generated update-status and adapter templates now include this MCP
doctor path. The MCP surface stays small: no new tool was added.

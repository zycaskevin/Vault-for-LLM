# Vault-for-LLM v0.6.53

This release moves memory automation one step closer to useful agent autonomy:
agents can now suggest memory cleanup and consolidation work, while formal
knowledge still stays behind the candidate review gate.

Highlights:

- Dream reports can produce candidate memory suggestions for stale, duplicate,
  isolated, or low-quality knowledge.
- `vault automation run --apply` can write Dream suggestions into the candidate
  queue when policy allows `dream_write_candidates`.
- Expired memories that are still useful or protected by privacy policy can now
  create `forgetting_suggestion` candidates instead of being silently skipped.
- Conservative automation remains report-only; balanced and autonomous modes can
  write candidates only when `--apply` is explicit.
- Repeated scheduled runs de-duplicate generated candidates by `source_ref`, so
  the review queue does not fill with the same suggestion again and again.

Smoke test:

```bash
pip install "vault-for-llm[mcp]==0.6.53"
vault automation run --project-dir /path/to/project --apply --json
```

Expected behavior: automation may create review candidates for Dream and
forgetting work, but it does not promote those candidates into active knowledge
or delete memories automatically.

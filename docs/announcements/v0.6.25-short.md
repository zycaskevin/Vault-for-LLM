# Vault-for-LLM v0.6.25

Vault-for-LLM v0.6.25 is a patch release focused on agent installation.

This version follows a real Hermes profile install smoke test. The core package
was already installing and running correctly, but the setup flow needed to be
more explicit for agents.

What changed:

- `vault setup-agent` now asks separate yes/no questions for MCP, semantic
  search, Supabase sync, Headroom context compression, and dev/benchmark
  dependencies.
- `vault setup-agent --project-dir ...` now uses that path as the installer
  target and creates it when missing.
- `vault remove <id> --confirm` and `vault delete <id> --confirm` are now
  available for reviewed knowledge deletion.
- Removing knowledge also cleans up related semantic, Document Map, lint, graph,
  FTS, and sqlite-vec rows.

The important UX change: agents should not silently skip Supabase, semantic
search, Headroom, or dev extras. They should ask the user, keep heavier extras
off by default, and only enable them when the workflow actually needs them.

Install:

```bash
pip install "vault-for-llm[mcp]==0.6.25"
vault setup-agent
```

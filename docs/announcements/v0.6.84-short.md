# Vault-for-LLM v0.6.84

This release makes multi-Agent startup easier to hand off.

`vault setup-agent` now writes adapter startup templates for:

- Codex
- Claude Code
- OpenClaw
- Hermes Agent

The generated files live under `agent-install/`:

- `README-agent-adapters.md`
- `codex-startup.md`
- `claude-code-startup.md`
- `openclaw-startup.md`
- `hermes-startup.md`
- `adapter-startup-contract.json`

The rule is intentionally small:

```text
update-status -> automation handoff -> search/read when needed -> propose candidate memory
```

The templates do not auto-upgrade runtimes, do not read raw transcripts by
default, and do not auto-promote memory.

```bash
vault setup-agent \
  --non-interactive \
  --agent codex \
  --scope shared \
  --agent-project-dir ~/Vaults/project-memory \
  --features core,mcp \
  --json
```

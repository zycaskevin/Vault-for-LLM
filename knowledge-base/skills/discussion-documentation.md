# 討論記憶與文檔化技能

## 用途
確保重要討論不會因記憶壓縮而遺失，提供可追溯的決策歷史

## 記憶類型

### 1. 決策記憶
- 每個關鍵決策的原因和依據
- 為什麼選擇 A 而不是 B

### 2. 細節記憶
- 討論中的具體約束和條件
- 不容易注意到的細節

### 3. 選項記憶
- 所有考慮過的選項及其優缺點
- 被拒絕的選項及原因

### 4. 上下文記憶
- 討論的背景和前提
- 與其他相關項目的關聯

## 文檔化策略

### 實時記錄
- 討論過程中立即記錄到文檔
- 不要等討論結束後再整理

### 結構化記錄
- 使用清晰的分類和標題
- 使用列表和表格提高可讀性

### 可追溯性
- 每個決策都有時間戳
- 記錄討論的參與者和角色

## 文檔模板

```markdown
# [項目名稱] 討論記錄

**日期**: YYYY-MM-DD
**參與者**: [角色1]、[角色2]、...
**討論主題**: [主題描述]

---

## 核心決策

### 決策 1: [決策描述]
**原因**: [為什麼這樣決定]
**選項比較**:
- 選項 A: [優點] / [缺點]
- 選項 B: [優點] / [缺點]
- 選項 C: [優點] / [缺點]
**結論**: [選擇了哪個選項及原因]

### 決策 2: [決策描述]
...

---

## 重要細節

- [細節 1]
- [細節 2]
...

---

## 約束和條件

- [約束 1]
- [約束 2]
...

---

## 實施路徑

Phase 1: [階段名稱]
├─ [任務 1]
├─ [任務 2]
└─ [任務 3]

Phase 2: [階段名稱]
...

---

## 下一步

- [待辦事項 1]
- [待辦事項 2]
...
```

## 使用場景

- 重要的項目討論
- 技術選型決策
- 架構設計會議
- 複雜需求分析

## 注意事項

1. 不要相信「我會記得」
2. 文檔要即時更新，不要等最後
3. 定期回顧文檔，確保沒有遺漏
4. 給文檔起有意義的名字，方便檢索

# Vault-for-LLM v0.6.68

This release makes session capture easier to start.

## What's New

- Added `vault capture discover`.
- Added MCP tool `vault_capture_discover` to the `review`, `maintenance`, and
  `full` profiles.
- Discovery ranks likely session transcript exports by filename, path, format,
  size, and source-system hints.
- Discovery does not read transcript contents.
- Result items include `capture_path`, which can be passed directly to
  `vault capture session` or `vault_capture_session`.
- The tool stays out of `core` so everyday MCP sessions remain small.

## Why It Matters

The session-memory loop is now two-step and agent-friendly:

```text
discover transcript -> preview capture -> write candidates -> inbox review -> promote or reject
```

This removes the brittle step where an agent has to guess where a Codex,
Hermes, OpenClaw, OpenCode, or Claude-style export was saved. The safety boundary
is unchanged: discovery is metadata-only, capture is preview-first, and active
memory still requires explicit promotion.

## Try It

```bash
pip install "vault-for-llm[mcp]==0.6.68"
vault capture discover --project-dir ~/Vaults/my-project --pretty
vault capture session sessions/codex-session.jsonl --project-dir ~/Vaults/my-project --pretty
```

For MCP:

```bash
vault-mcp --project-dir ~/Vaults/my-project --tool-profile review
```

Call `vault_capture_discover`, then pass a returned `capture_path` into
`vault_capture_session`.

# Vault-for-LLM v0.6.91

v0.6.91 makes the new low-risk memory auto-promote path installable by agents.

`vault setup-agent` now has:

```bash
--automation-auto-promote-low-risk
```

When enabled, setup writes `automation_policy.yaml` with
`auto_promote_low_risk_candidates: true`. Generated memory automation README
files also show the setting clearly, so the next agent can see whether the
closed loop is in preview-only mode or can promote low-risk candidates during
`--apply` runs.

The boundary stays narrow: low-risk auto-promote still requires the policy file,
`--apply`, passing privacy/duplicate/metadata/quality gates, a source reference,
and default `session_capture` / `session_lesson` / `low` sensitivity matching.

# Vault-for-LLM v0.6.59

This release makes scheduled memory automation smarter by default.

Agent-generated cron, LaunchAgent, and n8n templates now default to `vault automation cycle`. That means long-running agents can evaluate reviewed candidate outcomes, write bounded learning-policy hints, and then run safe memory maintenance in one scheduled pass.

The safety boundary stays the same: schedules remain report-first unless the user explicitly enables `--automation-apply`, and Vault still does not auto-promote candidates, hard-delete memory, or bypass privacy/access policy.

Users who want a simpler maintenance-only schedule can pass `--automation-command run`.

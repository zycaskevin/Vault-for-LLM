# Vault-for-LLM v0.6.31

This release turns the multi-agent memory-governance idea into real stored
metadata.

Highlights:

- Active knowledge and memory candidates now carry `scope`, `sensitivity`,
  `owner_agent`, `allowed_agents`, `memory_type`, and `expires_at`.
- CLI and MCP write paths can attach those fields through `vault add`,
  `vault remember`, `vault_add`, and `vault_memory_propose`.
- Promotion, Markdown compilation, Obsidian import/export, and Supabase sync now
  preserve the same governance fields instead of dropping them between systems.
- This gives Hermes, Codex, OpenClaw, Claude Code, Coze/n8n, and future agents a
  common metadata layer for private, project, shared, and public memory.

In plain language: Vault can now store not only "what should be remembered",
but also "who owns it, who may use it, how sensitive it is, and whether it
should expire."

# Vault-for-LLM v0.6.101

This release closes the smallest memory automation feedback loop.

`vault automation review-feedback` records whether one review-summary card was
accepted, rejected, or deferred. The event feeds the existing bounded learning
policy, so future review cards can be ranked by real outcomes.

Why it matters:

- humans still review only the small 5% surface
- agents can learn which review cards are useful
- rejected or deferred cards become training signals
- feedback does not promote, archive, delete, or mutate memory

Upgrade:

```bash
pip install -U "vault-for-llm[mcp]==0.6.101"
vault automation review-summary --write-summary
vault automation review-feedback --kind memory_importance --card-id 12 \
  --decision accept --reason "Correct card" --write-learning-policy
```

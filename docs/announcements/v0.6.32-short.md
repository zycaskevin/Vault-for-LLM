# Vault-for-LLM v0.6.32

This release makes governance metadata active on the read path.

Highlights:

- `vault search` and MCP `vault_search` now accept `agent_id`,
  `include_private`, and `max_sensitivity`.
- MCP `vault_map_show` and `vault_read_range` use the same policy, so an agent
  cannot bypass search filtering by reading a known knowledge ID directly.
- Without an explicit agent policy, local behavior stays backward-compatible.
- With an agent policy, `private` memory requires `include_private=true` plus
  owner or allow-list access; `restricted` memory requires owner or allow-list
  access.

In plain language: v0.6.31 stored who owns a memory and who may read it.
v0.6.32 starts using that information when agents search or read.

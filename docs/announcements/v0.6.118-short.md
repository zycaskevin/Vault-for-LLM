# Vault-for-LLM v0.6.118

This release continues the automation module cleanup by moving inbox and review-digest helpers into `vault.automation_inbox`.

The behavior is unchanged: `vault automation inbox` still returns a compact, read-only review queue; candidate content stays hidden by default; and inbox handoff files remain constrained under `reports/automation`.

This makes the automation core easier to audit while preserving the existing CLI and MCP automation inbox workflows.

# Vault-for-LLM v0.6.27

This is a follow-up release from real Hermes/Nancy installation testing.

Highlights:

- Fixed optional Supabase sync payloads so `content_hash` is always non-empty,
  even when older local rows are missing a hash.
- Added `--db` to `python -m scripts.sync_to_supabase` so scheduled jobs can
  target a stable `vault.db` path instead of depending on the current directory.
- Added `vault setup-agent --supabase-sync cron|launchagent|n8n|all` to generate
  daily Supabase sync templates.
- Added setup warnings when the current Python environment lives under `/tmp`,
  because temporary virtualenvs can disappear after reboot.
- Passed `fix_mistral_regex=True` when loading ONNX tokenizers to reduce
  tokenizer warning noise from recent Transformers versions.

Privacy gate failures remain intentional: if notes contain secrets, clean the
source notes and recompile instead of bypassing the gate.

# Vault-for-LLM v0.6.72

This release connects the compact cycle workspace to generated automation
schedules.

## What Changed

- Added `vault setup-agent --automation-write-workspace`.
- Added `--automation-workspace-inbox-limit` for scheduled cycle workspace size.
- Generated cron, LaunchAgent, and n8n templates can now run:

```bash
vault automation cycle \
  --write-workspace \
  --include-transcripts
```

## Why It Matters

Scheduled memory automation can now write both daily handoffs:

```text
scheduled cycle
  -> reports/automation/cycle-latest.json
  -> reports/automation/inbox-latest.json
```

The cycle workspace is the higher-level workbench: candidate review, optional
metadata-only transcript paths, and learning-policy summary. The inbox remains
the smaller review queue.

Safety stays the same: generated schedules are report-first unless
`--automation-apply` is explicitly enabled, transcript discovery does not read
transcript contents, and Vault still does not auto-promote candidates or
hard-delete memory.

## Try It

```bash
pip install "vault-for-llm[mcp]==0.6.72"

vault setup-agent \
  --non-interactive \
  --agent automation-agent \
  --scope shared \
  --agent-project-dir ~/Vaults/my-project \
  --features core,mcp,memory_agents \
  --automation-schedule cron \
  --automation-write-workspace \
  --automation-include-transcripts \
  --json
```

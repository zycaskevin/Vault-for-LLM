# v0.6.95 Short Announcement

Cold-store is now part of the automation loop.

```bash
vault automation run --apply --pretty
vault automation cycle --apply --write-workspace --pretty
```

When policy allows it, automation can now summarize expired-but-used memories,
archive them out of normal recall, and retain original content for audit or
restore. Reports, activity feeds, and automation briefs show cold-store preview,
applied, and skipped counts.

The safety boundary stays the same: no hard delete, private/high/restricted and
L0/L1 memories are skipped, and writes require `--apply`.

# Vault-for-LLM v0.6.45

This release hardens the remote Supabase reader path.

- Remote MCP map/read tools now verify the requested memory is readable before returning Document Map nodes, claims, or bounded source text.
- The Supabase policy template now exposes guarded RPCs for readable entries, nodes, claims, and content.
- Direct anon/authenticated reads from synced node and claim tables are revoked by the generated policy template.
- Remote search guidance now preserves `agent_id`, `include_private`, and `max_sensitivity` through the search -> map -> read workflow.

If you use Supabase remote readers, reapply `docs/supabase_read_policy.sql` before using `vault_remote_map_show` or `vault_remote_read_range`.

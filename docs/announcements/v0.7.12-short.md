## Vault-for-LLM v0.7.12

v0.7.12 adds optional signed MCP agent identity.

- Set `VAULT_MCP_REQUIRE_AGENT_SIGNATURE=1` to require HMAC-signed MCP calls.
- Configure `VAULT_MCP_AGENT_SECRET` for a shared local secret, or
  `VAULT_MCP_AGENT_SECRET_<AGENT>` for one agent.
- Signed calls include `agent_id` and `agent_signature`.
- Invalid signatures are rejected even when the require flag is off.

The feature is opt-in so existing local MCP clients keep working. Secrets stay
in environment variables and are not written into generated reports or startup
artifacts.

# 編程基本公理與原則
## 不可違反的核心規則
### 1. DRY 原則 (Don't Repeat Yourself)
**規則**：同一知識或邏輯在系統中只應出現一次。
**違反後果**：代碼冗餘、維護困難、一致性問題。
**執行方式**：提取重複代碼為函數、類或模塊。
### 2. KISS 原則 (Keep It Simple, Stupid)
**規則**：設計應盡可能簡單，避免不必要的複雜性。
**違反後果**：過度設計、理解困難、錯誤增加。
**執行方式**：優先選擇最簡單的可行方案。
### 3. YAGNI 原則 (You Ain't Gonna Need It)
**規則**：不要為未來可能需要的功能提前編寫代碼。
**違反後果**：代碼膨脹、維護負擔、潛在錯誤。
**執行方式**：只在真正需要時實現功能。

# Vault-for-LLM v0.6.66

This release connects the automation inbox to scheduled agents and MCP.

## What's New

- Added `vault automation inbox --write-handoff`.
- The handoff is written to `reports/automation/inbox-latest.json`.
- Generated cron, LaunchAgent, and n8n memory automation templates now run the
  selected `vault automation cycle|run` command and then write the inbox handoff.
- Added MCP tool `vault_automation_inbox` to the `review`, `maintenance`, and
  `full` tool profiles. It stays out of `core` to keep everyday MCP schemas
  small.

## Why It Matters

Scheduled memory automation now leaves a small next-agent handoff:

```text
scheduled cycle -> learning_policy + reports -> inbox-latest.json -> next agent review
```

Agents no longer need to parse full automation reports first. They can read the
compact inbox, decide which candidates need review, and only open larger reports
when necessary.

The safety boundary is unchanged: no auto-promotion, no hard delete, no
permission changes, and candidate content remains hidden by default.

## Try It

```bash
pip install "vault-for-llm[mcp]==0.6.66"
vault automation cycle --project-dir ~/Vaults/my-project --pretty
vault automation inbox --project-dir ~/Vaults/my-project --write-handoff --pretty
vault-mcp --project-dir ~/Vaults/my-project --tool-profile review
```

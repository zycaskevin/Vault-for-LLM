## 二、新發現的問題

### 🟠 High 級問題

#### 問題 N1：Hybrid/Semantic/Auto 模式在 `has_embeddings=False` 時無法使用語義索引
- **嚴重程度：** High
- **位置：** `vault/search.py` 第 1101-1138 行，`search` 方法
- **狀態：** ✅ 已在本次審查中修復
- **問題描述：**
  - `search` 方法在調用 `search_semantic` 或 `search_hybrid` 之前，會先檢查 `self.has_embeddings`
  - `has_embeddings` 屬性要求 `self._embed is not None` 且 `db._vec_available == True`
  - 但語義搜尋（`search_semantic`）使用的是 `semantic_vectors` 表，不需要 sqlite-vec 向量表
  - 當使用者只有語義索引（包括 hash 嵌入索引）而沒有 sqlite-vec 時，hybrid/semantic/auto 模式都會直接降級到關鍵詞搜尋
  - 導致測試 `test_search_qa_cli_hybrid_allow_hash_uses_semantic_index` 和 `test_search_qa_cli_auto_allow_hash_can_use_semantic_index` 失敗
- **修復方案：**
  - `semantic` 模式：改為優先檢查 `_semantic_index_available()`，其次 fallback 到 vector search，最後 fallback 到 keyword
  - `hybrid` 模式：檢查是否有任一第二來源（vector 或 semantic）可用，有則呼叫 `search_hybrid`
  - `auto` 模式：按優先級選擇（semantic hybrid > vector hybrid > keyword），不再要求 `has_embeddings` 作為前提
- **驗證結果：**
  - 修復前：2 個測試失敗
  - 修復後：全部 145 個測試通過
  - 無迴歸問題

### 🟡 Medium 級問題

#### 問題 N2：`vec_max_score` 變數計算但從未使用
- **嚴重程度：** Medium（程式碼壞味道）
- **位置：** `vault/search.py` 第 1833 行
- **問題描述：**
  - 在 `search_hybrid` 方法的動態權重調整部分
  - `vec_max_score = max(r.get('_score', 0) for r in second_results) if second_results else 0`
  - 這個變數被計算出來但從未被使用
  - 說明動態權重調整的實現不完整，只考慮了關鍵詞質量
- **建議修復：**
  -要嘛使用 `vec_max_score` 來優化動態權重調整邏輯
  - 要嘛刪除這個未使用的變數

#### 問題 N3：缺少配置參數驗證
- **嚴重程度：** Medium
- **位置：** `vault/search.py` 第 476-540 行，`__init__` 方法
- **問題描述：**
  - 構造函數有超過 20 個配置參數，但完全沒有參數驗證
  - 例如：`keyword_weight` 可以是負數，`query_expansion_count` 可以是負數
  - 無效參數可能導致難以排查的運行時錯誤
- **建議修復：**
  - 添加基本的參數範圍驗證
  - 對權重參數確保非負
  - 對數量參數確保 >= 0

#### 問題 N4：`_rerank` 靜態方法與實例方法職責不清
- **嚴重程度：** Medium
- **問題描述：**
  - 同時存在靜態方法 `_rerank` 和實例方法 `_rerank_with_strategy`
  - 靜態方法有自己的 `calc_freshness`、`calc_graph_depth` 實現（代碼重複）
  - 實例方法在某些情況下回退到靜態方法
  - 兩者的 rerank 演算法不完全一致，可能導致混淆
- **建議修復：**
  - 統一 rerank 邏輯到一個地方
  - 靜態方法保留為向後兼容的包裝器，內部呼叫統一的實現

### 🟢 Low 級問題

#### 問題 N5：註釋語言不一致
- **嚴重程度：** Low
- **問題描述：**
  - 部分註釋為中文，部分為英文
  - 代碼變數名為英文，但文件和註釋中英混雜
- **建議修復：**
  - 統一註釋語言（優先英文，或中英對照）

#### 問題 N6：缺少查詢擴展效果的對比測試
- **嚴重程度：** Low（提升質量）
- **問題描述：**
  - 沒有測試驗證查詢擴展確實能提升召回率
  - 沒有測試確保查詢擴展不會引入雜訊降低精確率
- **建議修復：**
  - 添加對比測試：有/無查詢擴展的召回率對比
  - 測試不同查詢類型的擴展效果

# 漸進式需求分析技能

## 用途
用於從願景到實施的漸進式需求分析，確保不遺漏關鍵細節

## 分析框架

### 第一層：願景層
- 目標是什麼？
- 價值是什麼？

### 第二層：架構層
- 核心框架是什麼？
- 各個模塊如何協作？

### 第三層：實施層
- 具體如何實現？
- 技術選型是什麼？

### 第四層：優化層
- 如何持續優化？
- 如何擴展和演化？

## 分析原則

1. 一次討論一層，避免跳躍
2. 每層都要與上層對齊
3. 每層都要有明確的交付物
4. 確認無誤後再進入下一層

## 使用示例

### hermes-dashboard 項目分析

#### 第一層：願景層
**目標**: 建立一個 24 小時運作的 AI 無人公司
**價值**: 可以自己研究、開發、審核、上架、收費；也可以作為商品教學或出售

#### 第二層：架構層
**核心框架**: Pipeline 系統驅動 5 個步驟
**協作**: 智能體團隊（CTO、開發員、Eve）通過 A2A 協議協作

#### 第三層：實施層
**智能體實現**: 持久化會話（常駐）+ 獨立進程（臨時）
**技術選型**: Next.js + Supabase + Redis

#### 第四層：優化層
**優化**: 自我優化機制（週期性 + 事件觸發）
**演化**: 動態技能樹，基於經驗成長

## 適用場景

- 新項目規劃
- 系統重構
- 產品需求分析
- 技術方案設計

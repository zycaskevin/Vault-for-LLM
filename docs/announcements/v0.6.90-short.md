# Vault-for-LLM v0.6.90

v0.6.90 adds the first safe path from memory candidate to active memory inside
the automation loop.

Automation still writes candidates by default. Formal memory promotion remains
off unless the user explicitly enables:

```yaml
auto_promote_low_risk_candidates: true
```

Even then, Vault only promotes candidates that match a narrow policy: session
capture source, session lesson type, low sensitivity, allowed scope, enough
trust, a source reference, and passing privacy, duplicate, metadata, and quality
gates.

Without `--apply`, Vault only previews what would be promoted. With `--apply`,
the report and cycle workspace show what was promoted and what stayed in the
review queue.

# Vault-for-LLM v0.6.76

This release wires the handoff command into generated agent setup files.

## What changed

- `vault setup-agent` memory automation templates now include a next-agent
  startup command:

```bash
vault automation handoff --project-dir /path/to/project
```

- `README-memory-automation.md` explains that this command is read-only and
  starts from `cycle-latest.md` when available.
- Setup next steps now remind agents to begin from the handoff after scheduled
  automation is enabled.

## Why it matters

Vault already had the closed-loop pieces:

1. scheduled automation writes compact handoffs
2. `vault automation handoff` reads the latest handoff
3. now `setup-agent` tells future agents to use that read path

That makes the automation loop easier to use in real agent sessions.

## Try it

```bash
pip install "vault-for-llm[mcp]==0.6.76"

vault setup-agent \
  --automation-schedule all \
  --automation-write-workspace
```

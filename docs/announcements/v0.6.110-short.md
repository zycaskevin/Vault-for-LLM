# Vault-for-LLM v0.6.110

This release adds a maintainability guardrail for the growing codebase.

## Highlights

- Added a baseline-based module-size gate.
- Added CI coverage so oversized modules cannot quietly grow.
- Documented the module-size rule in repo governance and a decision record.

## Why It Matters

Vault is becoming a full agent memory platform. As the system grows, large files make security review and feature work harder. This release does not force a disruptive refactor; it simply stops the largest modules from getting larger without an explicit review decision.

## Upgrade

```bash
pip install -U "vault-for-llm[mcp]==0.6.110"
```

No configuration migration is required.

# Vault-for-LLM v0.7.0rc2

v0.7.0rc2 is a release-candidate install-path polish release.

After testing the packaged v0.7.0rc1 `setup-agent` flow from PyPI, the generated startup pack worked, but `--agent-roster` roles were too easy to guess incorrectly. v0.7.0rc2 clarifies the allowed role values in CLI help and setup documentation:

`work`, `profile`, `care`, `dream`, `remote`, `automation`, and `observer`.

No new memory mutation path, hosted dependency, MCP permission, remote sync behavior, or background automation trigger was added.

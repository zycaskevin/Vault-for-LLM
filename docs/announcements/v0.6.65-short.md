# Vault-for-LLM v0.6.65

This release adds a compact review inbox for the automation memory loop.

## What's New

- Added `vault automation inbox`.
- The inbox reads pending/rejected memory candidates and the latest automation
  report, then returns the smallest useful review queue.
- It prioritizes privacy-blocked, sensitive, duplicate, weak-quality, and
  automation-generated candidates.
- Candidate content is hidden by default. Use `--include-content` only when an
  agent or reviewer explicitly needs redacted content.

## Why It Matters

The memory loop now has a daily review surface:

```text
capture -> candidates -> inbox -> review feedback -> learning policy -> Dream / forgetting -> next inbox
```

This keeps automation useful without turning memory maintenance into a giant
manual chore. The inbox is read-only: it does not promote candidates, delete
memory, or change permissions.

## Try It

```bash
pip install "vault-for-llm[mcp]==0.6.65"
vault automation inbox --project-dir ~/Vaults/my-project --limit 5 --pretty
vault capture session codex-session.jsonl --project-dir ~/Vaults/my-project --write-candidates
vault automation cycle --project-dir ~/Vaults/my-project --pretty
vault automation inbox --project-dir ~/Vaults/my-project --limit 5 --pretty
```

# Vault-for-LLM v0.6.99

This release connects explainable memory importance to the lifecycle loop.

`cold-store-expired`, `automation run`, and `automation cycle` now carry the
same `importance_score` used by `automation brief`. Expired-but-used memories
are sorted by importance before lifecycle review, and action ledgers explain the
score, signals, and recommendation.

Why it matters:

- review starts with the memories most likely to need refresh or summary
- cold-store decisions are easier to explain to humans and agents
- dashboards can show one shared lifecycle priority signal
- importance remains advisory; it does not bypass privacy, access policy,
  promotion rules, archive policy, or deletion safety

Upgrade:

```bash
pip install -U "vault-for-llm[mcp]==0.6.99"
vault automation run --pretty
```

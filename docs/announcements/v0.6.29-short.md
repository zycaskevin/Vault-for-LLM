# Vault-for-LLM v0.6.29

This release turns the multi-agent memory-governance design into something agents
can install, read, and review.

Highlights:

- Added memory governance docs for L0-L3 usage, user profile memory, shared vs
  private agent memory, Supabase RLS boundaries, and Obsidian frontmatter sync.
- Added the long-term vision doc: Vault is the user-owned memory core, agent
  runtimes are the hands, and model providers are replaceable compute tools.
- Added Progressive Memory Disclosure guidance so large lifelong memory vaults
  can stay efficient, permissioned, reviewable, and source-grounded.
- Added `vault setup-agent --features memory_agents`, which generates Profile /
  Dream / Forgetting agent guidance with report-only and candidate-only defaults.
- Added `vault candidates`, a CLI review queue for candidate memories that does
  not dump full raw content by default.

The main product direction is clearer now: Vault-for-LLM is not just storage for
agent notes. It is a local-first, candidate-first memory governance layer for
agent families.

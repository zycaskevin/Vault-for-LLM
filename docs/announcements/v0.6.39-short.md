# Vault-for-LLM v0.6.39

This patch hardens Supabase sync and agent installation guidance.

- Supabase sync now treats blank `content_hash` values as missing and regenerates
  a deterministic SHA256 hash before update/insert payloads are sent.
- Agent install prompts now ask for stable project and Python virtualenv paths,
  so long-lived installs do not accidentally depend on `/tmp`.
- README and agent runbook prompts were updated for agent-driven installs.

If an older sync run failed with a Supabase `content_hash` NOT NULL error, rerun
sync after upgrading:

```bash
python -m scripts.sync_to_supabase --db /path/to/project/vault.db --document-map --health
```

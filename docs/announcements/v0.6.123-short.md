# Vault-for-LLM v0.6.123

This release continues MCP module cleanup by moving local memory write/review/capture handlers into `vault.mcp_memory`.

## What Changed

- Split `vault_add`, `vault_memory_*`, and session capture/discovery handler logic out of the large `vault.mcp` module.
- Preserved existing `vault.mcp` helper import paths for compatibility.
- Updated the module-size baseline for `vault/mcp.py`.

## Compatibility

No MCP tool behavior changes are intended. Write policy checks, privacy gates, candidate-first memory review, and transcript path restrictions should match v0.6.122.

## Verification

The release was checked with focused MCP memory tests, module-size gates, release parity checks, README command smoke tests, public PR gates, full test suite, and a clean PyPI install smoke.

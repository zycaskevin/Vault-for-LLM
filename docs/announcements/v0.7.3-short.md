# Vault-for-LLM v0.7.3

v0.7.3 adds the first public shape of automatic, temporal, and reflective memory.

- `vault memory pipeline` discovers session transcripts, extracts reusable
  lessons, runs the existing gates, and writes candidate memories only when
  `--write-candidates` is explicit.
- Temporal metadata now separates fact validity from TTL: `valid_from`,
  `valid_until`, and `supersedes_id` let agents keep old facts for audit while
  distinguishing current facts.
- `vault memory temporal status/list` shows current, past, future, and timeless
  memories.
- `vault memory reflection` runs a report-first Dream plus lifecycle pass for
  consolidation, archive, and cold-store review.

Safety boundary:

- No automatic active-memory promotion is added.
- No hard deletion is added.
- Reflection remains report-first and candidate-first.
- Lifecycle changes still require the existing policy controls and explicit
  `--apply`.

# Vault-for-LLM v0.6.75

This release makes the automation handoff easier to consume.

## What changed

- Added `vault automation handoff`.
- By default it prints the latest readable handoff:
  - first `reports/automation/cycle-latest.md`
  - then `reports/automation/cycle-latest.json`
  - then `reports/automation/inbox-latest.json`
- Custom handoff paths must stay under `reports/automation`.

## Why it matters

The memory cycle can already produce a daily handoff. Now the next agent has a
single command for taking it in:

```bash
vault automation handoff --project-dir /path/to/project
```

That keeps the closed loop practical: scheduled jobs write the handoff, and the
next agent starts from the smallest useful context instead of searching through
full reports first.

## Try it

```bash
pip install "vault-for-llm[mcp]==0.6.75"

vault automation cycle \
  --project-dir /path/to/project \
  --write-workspace

vault automation handoff \
  --project-dir /path/to/project
```

# Vault-for-LLM v0.6.57

This release lets Dream and scheduled automation use the learning policy
created in v0.6.56.

`vault automation eval --write-learning-policy` can write bounded feedback
hints to `reports/automation/learning_policy.json`. v0.6.57 makes
`vault dream` and `vault automation run` read that file on later runs, annotate
matching candidate suggestions, and sort them by bounded priority.

## Highlights

- Dream suggestions now include learning metadata when a matching rule exists.
- Candidate suggestions can be sorted by bounded learning priority.
- Automation report summaries include Dream learning-policy status and applied
  rule counts.
- `vault automation report --latest` ignores `learning_policy.json`, so handoff
  files are not mistaken for timestamped run reports.

The learning policy remains guidance only. It does not auto-promote candidates,
delete memory, or bypass privacy and access policy.

## Install

```bash
pip install "vault-for-llm[mcp]==0.6.57"
```

## Try It

```bash
vault automation eval --write-learning-policy --pretty
vault dream --mode report --write-report --pretty
```

# Vault-for-LLM v0.6.30

This release closes the candidate-memory review loop for MCP agents.

Highlights:

- Added `vault_memory_candidates`, an MCP tool for reviewing pending memory
  candidates without reading SQLite directly.
- The tool defaults to pending `candidate` rows and returns compact previews
  instead of full raw content.
- Trusted review agents can request `include_gates=true` for the full gate
  payload, and `include_content=true` only when a preview is not enough.
- The `review` and `maintenance` MCP tool profiles now include
  `vault_memory_candidates`; the small `core` profile stays unchanged.

This means CLI users can run `vault candidates`, while MCP users can review the
same candidate queue through `vault_memory_candidates` before promoting memory.

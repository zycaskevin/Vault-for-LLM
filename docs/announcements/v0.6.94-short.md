# v0.6.94 Short Announcement

This release adds the first explicit long-term forgetting action:

```bash
vault usage cold-store-expired --pretty
vault usage cold-store-expired --apply --pretty
```

It is for expired memories that are still retrieved or cited. Instead of simply
archiving them, Vault writes a compact summary, moves the row out of normal
recall with `status=archived`, and keeps the original content in `vault.db` for
audit or restore.

MCP maintenance agents can preview or apply the same workflow with:

```text
vault_cold_store_expired
```

The MCP tool is not in the `core` profile. It defaults to dry-run and skips
private, high/restricted, and L0/L1 memories.

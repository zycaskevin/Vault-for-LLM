# Vault-for-LLM v0.6.35

This release adds a CLI path for Supabase remote memory reads.

What's new:

- Added `vault remote search`, `vault remote map`, and `vault remote read`.
- The CLI remote search uses the same Supabase `vault_search_readable` RPC as MCP `vault_remote_search`.
- n8n, cron jobs, shell scripts, and non-MCP agents can now use the remote workflow directly.
- The intended flow is `vault remote search` -> `vault remote map` -> `vault remote read`.
- Remote search returns safe summaries and metadata; bounded reads still require synced Document Map rows and claims/content.

Vault remains local-first. Use this only when you intentionally configured Supabase as a cross-host or hosted-agent read target.

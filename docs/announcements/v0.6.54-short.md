# Vault-for-LLM v0.6.54

This release makes agent installation easier to verify after setup.

The key change is simple: when an agent runs `vault setup-agent`, Vault now
generates a local smoke-test script for that exact project vault. The script
checks the local path before the user spends time configuring Supabase, n8n,
Coze, Obsidian, or other optional integrations.

Highlights:

- `vault search` now supports `--json` and `--pretty` for reliable agent-side
  validation.
- `vault setup-agent` now writes `agent-install/local-smoke.sh`.
- The local smoke script verifies `vault add`, structured search results,
  `vault remember --mode candidate`, and candidate listing.
- The smoke script does not promote candidates or delete memories.
- Agent install docs now recommend running the local smoke test before remote
  setup.

Smoke test:

```bash
pip install "vault-for-llm[mcp]==0.6.54"
vault setup-agent --non-interactive --agent codex --scope shared --features core,mcp --json
sh ~/Vaults/project-memory/agent-install/local-smoke.sh
```

Expected behavior: the script exits successfully with `local_smoke=ok`, proving
the configured local vault can write, search, and create reviewable memory
candidates.

# Vault-for-LLM v0.6.77

This release adds the first local multi-agent registry.

When an Agent runs `vault setup-agent`, Vault now records that runtime in:

```text
~/.vault-for-llm/agent-registry.json
```

Agents can then start with:

```bash
vault update-status
```

The status output shows:

- installed Vault version
- optional latest-version comparison
- registered Agents on this machine
- project vault paths
- startup handoff commands such as `vault automation handoff --project-dir ...`

Manual registration is also available:

```bash
vault agent register --agent codex --project ~/Vaults/my-project --scope shared
vault agent list
vault agent status --latest-version 0.6.77
```

This is a foundation for the hybrid multi-agent model: one shared local Vault
runtime, shared project vaults, private Agent vaults later, and optional
Supabase sync for cross-device memory.

Install:

```bash
pip install "vault-for-llm[mcp]==0.6.77"
```

# Vault-for-LLM v0.6.106

This release makes generated Agent startup templates fleet-aware.

`vault setup-agent` now writes startup contracts that tell MCP and CLI agents how
to consume the v0.6.105 handoff shape:

1. read update status;
2. read `fleet_health_content` first when it exists;
3. read the selected cycle/inbox `content`;
4. search and bounded-read only when needed;
5. propose durable lessons as candidates.

Generated files updated:

- `mcp-startup.json`
- `README-mcp-startup.md`
- `adapter-startup-contract.json`
- `README-agent-adapters.md`
- `runtime-update-playbook.json`
- runtime templates for Codex, Claude Code, OpenClaw, and Hermes Agent

Safety boundary: the templates remain read-only at startup. They do not
auto-upgrade runtimes, read raw transcripts by default, or auto-promote memory.

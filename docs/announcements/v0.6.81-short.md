# Vault-for-LLM v0.6.81

This release makes local multi-Agent update status easier to coordinate.

## What Changed

- `vault update-status` now includes `agent_update_notices`.
- MCP `vault_update_status` returns the same per-Agent notices.
- Each notice shows the registered Agent version, project vault path, private
  vault path when present, and whether that Agent may need an upgrade or
  restart.
- Human-readable `vault update-status` output prints those notices directly.

## Why It Matters

Many users run several Agent runtimes on the same computer, often in different
virtualenvs. One runtime may update Vault before the others.

With this release, the local registry becomes a small machine-level noticeboard:
run `vault update-status --write-status` once, and other local runtimes can read
the same status through CLI or MCP before they start work.

## Install

```bash
pip install "vault-for-llm[mcp]==0.6.81"
vault setup-agent
```

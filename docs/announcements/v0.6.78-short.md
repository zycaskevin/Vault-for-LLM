# Vault-for-LLM v0.6.78

This release adds the first hybrid vault layout to `setup-agent`.

Agents can now install with:

```bash
vault setup-agent \
  --non-interactive \
  --agent codex \
  --scope shared \
  --agent-project-dir ~/Vaults/project-memory \
  --memory-layout hybrid \
  --features core,mcp \
  --json
```

Hybrid layout means:

- shared project vault: reviewed project knowledge, SOPs, fixes, releases,
  benchmarks, and safety rules
- private Agent vault: identity, preferences, personal notes, and
  agent-specific working style

`setup-agent` writes:

```text
agent-install/hybrid-vault-layout.json
agent-install/README-hybrid-vault-layout.md
```

The local registry also records the private Agent vault path, so
`vault update-status` can show both shared and private memory locations.

This is storage coordination, not automatic cross-vault search yet. Private
Agent memory stays local-only by default.

Install:

```bash
pip install "vault-for-llm[mcp]==0.6.78"
```

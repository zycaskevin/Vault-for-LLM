# Vault-for-LLM v0.6.89

This release closes one more part of the memory automation loop.

## What changed

- `vault automation cycle --capture-transcripts --apply` can now turn discovered
  session transcripts into gated memory candidates.
- The generated cycle workspace records transcript-capture status, counts, and
  candidate IDs while keeping raw transcript and candidate content hidden.
- `vault setup-agent --automation-capture-transcripts` can add the same opt-in
  behavior to generated cron, LaunchAgent, and n8n templates.

## Safety boundary

Transcript discovery is still metadata-only. Transcript capture reads selected
files only when explicitly enabled and `--apply` is present. It writes
candidates only; it never promotes active memory, hard-deletes memory, or
bypasses privacy gates.

## Smoke test

```bash
vault automation cycle \
  --project-dir /path/to/project \
  --apply \
  --include-transcripts \
  --capture-transcripts \
  --write-workspace \
  --json
```

Expected: session lessons enter `memory_candidates`, and
`reports/automation/cycle-latest.md` shows a transcript-capture summary without
raw transcript content.

# Decision Record: Promote v0.7 to Stable

Date: 2026-06-25
Version: v0.7.0

## Context

The v0.7 release-candidate line validated the Agent Knowledge Platform boundary:

- v0.7.0rc1 proved PEP 440 release-candidate publishing, package parity, and the platform-boundary documentation path.
- v0.7.0rc2 fixed setup-agent roster role clarity after a packaged install smoke found that free-form roles such as `engineering` were too easy to guess incorrectly.
- Clean PyPI install smoke verified MCP memory and automation routing.
- Packaged `setup-agent` smoke verified hybrid shared/private layout, startup templates, Supabase advanced setup guidance, remote reader templates, validation pack generation, automation schedule templates, stable venv script generation, and `startup-doctor`.
- README minimal-path smoke verified a simple install without Supabase credentials or Obsidian.

## Decision

Promote the v0.7 line to stable `0.7.0`.

The stable release should communicate that Vault-for-LLM is now an Agent Knowledge Platform foundation, not merely a pile of CLI commands.

## Smoke-Test Clarification

`vault remember` is candidate-first by design. A remembered candidate should not appear in normal active-memory search until it is reviewed and promoted.

Therefore install smoke tests should separate:

- active search/read check: create a temporary active note with `vault add`
- propose check: create a candidate with `vault remember`

## Safety Notes

v0.7.0 does not add a new memory mutation path, hosted dependency, MCP permission, auto-promote path, remote sync behavior, or background automation trigger beyond the validated RC line.

Stable v0.7.0 follows clean package install, Agent setup smoke, startup doctor, MCP smoke, privacy scan, module-size gate, README command smoke, Search QA gate, and release parity validation.

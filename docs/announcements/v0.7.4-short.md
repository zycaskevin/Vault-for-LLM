# Vault-for-LLM v0.7.4

v0.7.4 is a small installer-smoke patch for the v0.7.3 automatic, temporal, and reflective memory release.

What changed:

- Generated `agent-install/local-smoke.sh` now derives its default Python
  interpreter from the installed `vault` console script when `PYTHON` is not
  explicitly set.
- This keeps setup-agent self-tests aligned with the venv or packaged
  environment that actually owns Vault-for-LLM.

No new memory mutation path, cloud dependency, MCP permission, or background
automation trigger is added.

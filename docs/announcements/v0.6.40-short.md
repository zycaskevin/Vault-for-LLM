# Vault-for-LLM v0.6.40

This release turns stable agent virtualenv setup into an installer artifact.

- `vault setup-agent --stable-venv PATH` now writes
  `agent-install/setup-stable-venv.sh` and `README-stable-venv.md`.
- `vault setup-agent --write-stable-venv-script` uses the default long-lived
  path `~/.hermes/venvs/vault-for-llm`.
- The generated script creates a stable venv, installs the selected
  Vault-for-LLM extras, installs Headroom when selected, and runs the matching
  non-interactive setup command.

Use this when an agent was installed under `/tmp/...` and scheduled jobs,
Supabase sync, or MCP commands need a reboot-safe Python environment.

# Vault-for-LLM v0.7.8

This patch makes the next-agent startup path smaller and more automatic.

`vault automation handoff` and MCP `vault_automation_handoff` now attach the
latest review-summary card deck and learning-health dashboard when those files
exist. Agents still get the original selected handoff in `content`, but now they
also receive:

- `fleet_health_content`
- `review_summary_content`
- `learning_health_content`
- `content`

Generated setup-agent startup packs and `startup-doctor` now use that same read
order, so Codex, Claude Code, OpenClaw, Hermes Agent, and other runtimes can
start from one shared contract.

Safety stays the same: these startup prefaces are read-only. They do not promote
memory, archive rows, delete rows, read raw transcripts, or apply lifecycle
actions.

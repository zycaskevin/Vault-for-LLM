# Vault-for-LLM v0.6.109

This release is a small security-maintenance cut.

## Highlights

- MCP rate limiting now lives in `vault.mcp_security`.
- MCP write-governance checks now live in the same security helper module.
- Public MCP tool names and payloads are unchanged.

## Why It Matters

The previous release added stronger write boundaries for shared multi-agent vaults. This release makes that boundary easier to maintain, test, and reuse as more MCP tools are added.

## Upgrade

```bash
pip install -U "vault-for-llm[mcp]==0.6.109"
```

No configuration migration is required.

# Vault-for-LLM v0.6.124

v0.6.124 is another 0.7-readiness cleanup release.

The MCP server now routes automation inbox, activity, brief, handoff, dream, and cold-store lifecycle handlers through `vault.mcp_automation`. Public tool names and JSON payloads stay the same.

This keeps `vault/mcp.py` moving toward a thin router while leaving automation safety defaults unchanged: no new tool, write permission, remote sync path, or background automation trigger was added.

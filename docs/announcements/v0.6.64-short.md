# Vault-for-LLM v0.6.64

This release hardens the first session-capture entry point for memory automation.

## What's New

- Added `vault capture session` for JSONL, Markdown, and plain-text agent
  transcripts.
- The extractor looks for reusable decisions, pitfalls, workflows, and
  source-of-truth lines.
- The default mode is dry-run preview. Use `--write-candidates` to write
  captured lessons into the memory candidate queue.
- Captured items go through the existing privacy, duplicate, metadata, and
  quality gates.
- Standalone `sk-...` API-key shaped tokens are now treated as fail-severity
  secrets even when a transcript does not include an `api_key =` label.

## Why It Matters

Automation now has an ingestion path: after Codex, Hermes, OpenClaw, Claude
Code, or another agent finishes a work session, the transcript can become
reviewable candidate memory without promoting anything automatically.

This keeps the safety boundary intact:

- no auto-promotion
- no hard delete
- no permission changes
- active memory remains review-first

## Try It

```bash
pip install "vault-for-llm[mcp]==0.6.64"
vault capture session codex-session.jsonl --project-dir ~/Vaults/my-project --pretty
vault capture session codex-session.jsonl --project-dir ~/Vaults/my-project --write-candidates
vault candidates --project-dir ~/Vaults/my-project --include-gates --pretty
```

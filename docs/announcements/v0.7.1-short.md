# Vault-for-LLM v0.7.1

v0.7.1 is a small stability patch for the first stable v0.7 line.

It does not add a new hosted dependency, background automation trigger, MCP permission, or memory mutation path. It tightens the install and smoke-test experience after the v0.7.0 PyPI closeout found two easy-to-miss CLI details:

- active notes use `vault add "Title" --content "..."`
- bounded source reads use `vault map read <knowledge_id> --lines START-END`

The README quickstart now shows both shapes, the README command smoke verifies `map build/read`, and generated `agent-install/local-smoke.sh` now checks active add/search, Document Map bounded read, candidate-first `remember`, candidate listing, and core MCP startup tools.

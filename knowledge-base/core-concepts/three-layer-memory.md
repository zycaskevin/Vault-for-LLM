# 三層記憶架構

## 概述
Hermes 採用三層記憶架構，實現智能的上下文管理。

## 三層結構

### Layer 1: 實時緩存（實時對話）
- **用途**: 當前對話上下文
- **特點**: 高速訪問，會話結束後清除
- **實現**: 內存緩存

### Layer 2: 短期記憶（每日記錄）
- **用途**: 每日記錄和日誌
- **特點**: 持久化存儲，按天組織
- **實現**: 文件系統 (memory/YYYY-MM-DD.md)

### Layer 3: 長期知識（蒸餾記憶）
- **用途**: 重要事實、偏好、決策
- **特點**: 結構化存儲，可搜索
- **實現**: Vector DB + Metadata

## 優勢
- 按需加載，減少 token 消耗
- 結構化組織，提高召回效率
- 分層管理，平衡性能和成本

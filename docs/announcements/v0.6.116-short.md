# Vault-for-LLM v0.6.116

This release continues the setup-agent module split by moving Supabase setup guide rendering and advanced read-policy SQL into `vault.agent_setup_supabase`.

The public installer behavior is unchanged: `vault setup-agent` still generates the same Supabase guide and `supabase-read-policy.sql`, and existing imports from `vault.agent_setup` remain compatible.

This keeps the installer easier to review while preserving the agent-friendly setup path for local-only, simple Supabase sync, and advanced multi-agent/RLS deployments.

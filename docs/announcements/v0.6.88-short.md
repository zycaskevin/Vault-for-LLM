# Vault-for-LLM v0.6.88

This release makes generated runtime templates easier for agents to install
safely.

New command:

```bash
vault agent install-runtime-template --runtime codex --target ./AGENTS.md
vault agent install-runtime-template --runtime codex --target ./AGENTS.md --apply
```

The command is dry-run by default. With `--apply`, Vault writes a marked startup
block and creates a backup before changing an existing file. Re-running the
command replaces the same block instead of duplicating instructions.

This helps Codex, Claude Code, OpenClaw, Hermes Agent, and other local runtimes
share one Vault project without forcing users to manually copy generated
templates around.

Also included:

- a cross-runtime update-notice smoke test for four runtimes on one shared
  project vault;
- a shorter README install path, with detailed multi-runtime behavior moved to
  `docs/agent_install.md`.

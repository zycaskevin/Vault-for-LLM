# Vault-for-LLM v0.6.85

This release closes the local multi-Agent update loop.

When one runtime upgrades Vault-for-LLM, the others need a shared local signal
that says whether they should restart, re-register, or upgrade too.

New commands:

```bash
vault agent doctor
vault update-status --doctor
```

The doctor checks:

- `~/.vault-for-llm/update-status.json` exists
- the notice is fresh enough
- every registered Agent appears in `agent_update_notices`
- no registered Agent is behind the current runtime or latest known version

`vault setup-agent` now also writes:

- `agent-install/refresh-update-status.sh`
- `agent-install/README-agent-update-rollout.md`

After upgrading one runtime, run:

```bash
sh agent-install/refresh-update-status.sh
```

Then each local Agent can read its focused startup view:

```bash
vault update-status --read-status --agent codex --json
```

This is still not an auto-upgrader. It is a shared local notice and health
check so multiple runtimes can coordinate without fragmenting memory.

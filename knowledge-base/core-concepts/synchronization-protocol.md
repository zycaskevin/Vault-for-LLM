# 同步協定 (Synchronization Protocol)

> 類型: knowledge | 類別: core-concept | 來源: nancy
> 
> 提交時間: 2026-04-08T13:10:00Z

## 概述

定義多龍蝦環境下知識庫的同步策略和衝突解決機制。

---

## 同步觸發條件

| 觸發條件 | 動作 | 模式 |
|----------|------|------|
| 定時 (每60分鐘) | pull 最新知識 | Light |
| 任務完成 | push 經驗/錯誤 | Full |
| 新技能發現 | push skill | Full |
| 錯誤發生 | push error | Full |
| 手動請求 | full sync | Full |

## 同步流程

```
開始同步
  ↓
1. git fetch origin
  ↓
2. 檢查本地 vs 遠端差異
  ↓
3. 衝突檢測
  ├─ 無衝突 → git pull --ff-only → 完成
  └─ 有衝突 → 進入衝突解決流程
  ↓
4. 更新本地索引
  ↓
5. 生成同步報告
```

## 衝突解決策略

### 策略 1: 自動合併
- 不同檔案 → 自動合併
- 同檔案不同段落 → 自動合併
- 同段落衝突 → 進入人工審核

### 策略 2: 優先級
- CEO 更新 > 其他龍蝦
- 錯誤記錄 > 經驗記錄
- 最新提交 > 舊提交

### 策略 3: 通知
衝突發生時通知 Arthur 進行仲裁。

## 狀態標記

| 狀態 | 標記 | 說明 |
|------|------|------|
| 已同步 | 🟢 | 所有龍蝦一致 |
| 待推送 | 🟡 | 本地有未推送內容 |
| 待拉取 | 🔵 | 遠端有新內容 |
| 衝突 | 🔴 | 需要人工解決 |

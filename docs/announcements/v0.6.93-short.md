# v0.6.93 Short Announcement

This release adds one small daily surface for the automation loop:

```bash
vault automation brief --pretty
```

The brief joins five signals into one read-only view:

1. what the system is learning from promote/reject feedback;
2. which memories are used or cited often;
3. which expired memories should be archived, summarized, or reviewed;
4. which local agents are registered for the shared vault;
5. the shortest 5% human-review queue.

MCP agents can call the same view with:

```text
vault_automation_brief
```

The tool is in the `core` MCP profile. It does not promote memory, expose raw
candidate content, compress memories, or move anything to cold storage. It is a
startup and dashboard view for deciding what needs attention next.

# Strategy: Agent Memory Governance

Vault-for-LLM should not be positioned as another RAG database, note app, or
Obsidian plugin. The product bet is narrower and stronger:

> Vault-for-LLM is the memory governance layer for multi-agent teams.

It helps operators decide what agents may remember, trust, share, forget, and
roll back.

This strategy folder turns that positioning into durable product guidance:

- [Positioning](positioning.md): the category, failure modes, and core language.
- [Product Architecture](product-architecture.md): OSS core, self-host team,
  hosted cloud, and enterprise governance layers.
- [Killer Demo](killer-demo.md): the shared-memory workflow that proves the
  difference from ordinary RAG.
- [90-Day Validation](90-day-validation.md): concrete traction gates before
  building heavier platform surfaces.
- [Enterprise Feature Map](enterprise-feature-map.md): paid governance
  capabilities and what stays in open source.

Use these documents before large roadmap, integration, pricing, cloud, or
enterprise decisions.

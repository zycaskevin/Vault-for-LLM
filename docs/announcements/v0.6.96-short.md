# v0.6.96 Short Announcement

Vault-for-LLM v0.6.96 tightens the human-review surface for the automation
loop.

The main change is `automation inbox` review digest cards. Instead of asking a
person to inspect a long JSON report first, Vault now turns report-level review
signals into short cards with a recommended action and a safe action:

- protected expired memory;
- expired-but-used memory;
- cold-store summaries;
- auto-promote previews;
- Dream and Forgetting suggestions.

`automation brief` uses the same digest for its 5% human-review section. The
goal is simple: agents should keep doing the routine memory work, while humans
only see the few decisions that actually need judgment.

```
│  │   └── Topics（主題）                                     │

│  └── 🚪 Room（房間）— 子主題                                │

│      │   └── deploy（部署）                                 │
```

```
│                    AAAK 壓縮格式                              │

│  │ TEAM: PRI(lead) | KAI(backend,3yr) SOR(frontend)   │   │
│  │       MAY(infra) LEO(junior,new)                   │   │
│  │ PROJ: DRIFTWOOD(saas.analytics) | SPRINT: auth    │   │
│  │ DECISION: KAI.rec:clerk>auth0(pricing+dx) | ★★★★ │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  特點：人類可讀 + LLM 可解析 + 30x 壓縮率                   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 四層載入機制

```
┌─────────────────────────────────────────────────────────────┐
│                    分層載入機制                              │
├──────────────┬────────────────┬─────────────────────────────┤
│    層次       │     內容        │         觸發條件            │
├──────────────┼────────────────┼─────────────────────────────┤
│  L0 身份      │ AI 基本設定     │ 永遠載入（~50 tokens）     │
│  L1 關鍵事實  │ 團隊、專案      │ 永遠載入（~120 tokens）    │
│  L2 房間召回  │ 最近 session    │ 按需求（主題浮現時）       │
│  L3 深度搜尋  │ 語義搜尋        │ 明確請求時                 │
└──────────────┴────────────────┴─────────────────────────────┘

成本對比：
- LLM Summaries：~$507/年（650K tokens）
- MemPalace：~$0.70/年（170 tokens）
```

```python
# Temporal Knowledge Graph（SQLite）
kg.add_triple("Kai", "works_on", "Orion", valid_from="2025-06-01")

# → [Maya → assigned_to → auth-migration (active)]
```

## 📚 LLM Wiki 架構分析

### 三層架構

```
┌─────────────────────────────────────────────────────────────┐
│                    LLM Wiki 三層架構                        │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  ┌─────────────────────────────────────────────────────┐  │
│  │  Layer 1: Raw Sources（原始資料）                    │  │
│  │  ─────────────────────────────────────────────────  │  │
│  │  • 不可變的事實來源                                 │  │
│  │  • 文章、論文、圖片                                 │  │
│  │  • LLM 只讀取，不修改                              │  │
│  └─────────────────────────────────────────────────────┘  │
│                          ↓ Ingest                          │
│  ┌─────────────────────────────────────────────────────┐  │
│  │  Layer 2: Wiki（維基層）                           │  │
│  │  ─────────────────────────────────────────────────  │  │
│  │  • LLM 生成的 Markdown 檔案                       │  │
│  │  • 摘要、實體頁面、概念地圖                        │  │
│  │  • LLM 完全擁有此層                                │  │
│  └─────────────────────────────────────────────────────┘  │
│                          ↑ Lint                           │
│  ┌─────────────────────────────────────────────────────┐  │
│  │  Layer 3: Schema（結構定義）                        │  │
│  │  ─────────────────────────────────────────────────  │  │
│  │  • CLAUDE.md / AGENTS.md                           │  │
│  │  • 定義 wiki 結構、慣例、工作流                    │  │
│  │  • 讓 LLM 成為「有紀律的維護者」                  │  │
│  └─────────────────────────────────────────────────────┘  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 工作流程

```
┌─────────────────────────────────────────────────────────────┐
│                    LLM Wiki 工作流程                        │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  📥 Ingest（新資料入庫）                                    │
│  ├── 1. LLM 讀取新來源                                      │
│  ├── 2. 與使用者討論要點                                    │
│  ├── 3. 撰寫摘要頁面                                       │
│  ├── 4. 更新全域索引與相關頁面                              │
│  └── 5. 附加到時間日誌                                      │
│                                                              │
│  🔍 Query & Synthesis（查詢與合成）                          │
│  ├── 1. LLM 搜尋 wiki                                      │
│  ├── 2. 讀取相關頁面                                        │
│  ├── 3. 合成答案                                            │
│  └── 4. 有價值的答案寫回 wiki（複合效應）                   │
│                                                              │
│  ✅ Lint（健康檢查）                                        │
│  ├── 1. 檢查矛盾（新舊資料衝突）                           │
│  ├── 2. 檢查孤兒頁面（無 inbound links）                   │
│  └── 3. 檢查過時聲明或缺失 cross-references                 │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 導航機制

```
┌─────────────────────────────────────────────────────────────┐
│                    導航檔案結構                              │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  📋 index.md — 內容分類目錄                                  │
│  │   LLM 首先讀取此檔案找到相關頁面                          │
│  │   不需要複雜向量資料庫（~100-500 sources 規模）          │
│  │                                                          │
│  📅 log.md — 時間日誌（僅附加）                             │
│  │   格式：## [YYYY-MM-DD] ingest | Title                  │
│  │   方便 grep 解析                                        │
│  │                                                          │
│  📖 CLAUDE.md — Schema 定義                                │
│      定義 wiki 結構、慣例、攝取/查詢/檢查工作流             │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 🏗️ Guardrails 現有架構（我們的系統）

### 五層融合架構

```
┌─────────────────────────────────────────────────────────────┐
│                 Guardrails 五層融合架構                       │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  ┌─────────────────────────────────────────────────────┐  │
│  │  Layer 1: Raw（原始層）                              │  │
│  │  ─────────────────────────────────────────────────  │  │
│  │  • 原始資料不入庫                                   │  │
│  │  • 來源：web-clips, system-logs, agent-outputs    │  │
│  │  • 格式：Markdown + YAML frontmatter               │  │
│  └─────────────────────────────────────────────────────┘  │
│                          ↓ 增量編譯                        │
│  ┌─────────────────────────────────────────────────────┐  │
│  │  Layer 2: Compiled（編譯層）                        │  │
│  │  ─────────────────────────────────────────────────  │  │
│  │  • 結構化知識                                       │  │
│  │  • 分類：techniques, concepts, workflows, axioms  │  │
│  │  • 包含：摘要、標籤、相關連結                      │  │
│  └─────────────────────────────────────────────────────┘  │
│                          ↓ 品質控制                        │
│  ┌─────────────────────────────────────────────────────┐  │
│  │  Layer 3: Quality（質量層）                         │  │
│  │  ─────────────────────────────────────────────────  │  │
│  │  • 信賴度評分                                       │  │
│  │  • 衝突偵測                                         │  │
│  │  • 人工標記                                         │  │
│  └─────────────────────────────────────────────────────┘  │
│                          ↓ 應用                            │
│  ┌─────────────────────────────────────────────────────┐  │
│  │  Layer 4: Application（應用層）                     │  │
│  │  ─────────────────────────────────────────────────  │  │
│  │  • 搜尋介面                                        │  │
│  │  • 圖譜視覺化（Graphify）                          │  │
│  │  • API 端點                                        │  │
│  └─────────────────────────────────────────────────────┘  │
│                          ↓ 協作                            │
│  ┌─────────────────────────────────────────────────────┐  │
│  │  Layer 5: Collaboration（協作層）                    │  │
│  │  ─────────────────────────────────────────────────  │  │
│  │  • Git 同步                                        │  │
│  │  • 跨 Agent 共享                                  │  │
│  │  • Nancy/Eve/Coco 共用同一知識中樞               │  │
│  └─────────────────────────────────────────────────────┘  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 現有缺失分析

```
┌─────────────────────────────────────────────────────────────┐
│                    Guardrails 現有缺失                        │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  ⚠️ 缺失一：無分層載入機制                                 │
│     → 所有資料同等載入，context 膨脹                        │
│     → 需要類似 L0-L3 的分層機制                            │
│                                                              │
│  ⚠️ 缺失二：無壓縮格式                                     │
│     → 原始內容直接存儲，佔用空間大                         │
│     → 需要類似 AAAK 的壓縮格式                             │
│                                                              │
│  ⚠️ 缺失三：無時間維度追蹤                                │
│     → 無法查詢「某時間點的狀態」                          │
│     → 需要類似 Temporal Knowledge Graph                    │
│                                                              │
│  ⚠️ 缺失四：無 Schema 驅動                                 │
│     → 缺乏明確的結構定義檔案                              │
│     → 需要類似 CLAUDE.md 的 schema                         │
│                                                              │
│  ⚠️ 缺失五：無 Lint 機制                                   │
│     → 無自動矛盾偵測                                       │
│     → 無孤兒頁面檢查                                       │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

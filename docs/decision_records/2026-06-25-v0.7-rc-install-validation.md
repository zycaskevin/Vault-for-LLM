# Decision Record: v0.7 RC Install Validation

Date: 2026-06-25
Version: v0.7.0rc2

## Context

v0.7.0rc1 was published as the first Agent Knowledge Platform release candidate. After publishing, the packaged PyPI path was tested with a non-interactive `setup-agent` run that generated:

- hybrid shared/private memory layout
- Codex, Claude Code, OpenClaw, and Hermes startup templates
- MCP startup contract files
- Supabase advanced setup guidance
- remote reader templates
- live validation pack
- automation cron schedule templates
- stable venv bootstrap script

The generated startup pack passed `vault agent startup-doctor`.

## Finding

The package worked, but one user-facing edge was easy to trip:

```bash
--agent-roster codex:engineering
```

failed because roster roles are intentionally constrained. The allowed roles are:

- `work`
- `profile`
- `care`
- `dream`
- `remote`
- `automation`
- `observer`

The validation rerun using `codex:work,hermes:care,openclaw:work,claude-code:work` generated the expected install pack and passed startup doctor.

## Decision

Ship v0.7.0rc2 as an install-path clarity release.

Do not loosen roster roles into arbitrary strings yet. The fixed role set keeps access-matrix defaults deterministic and easier to review.

## Safety Notes

This RC does not add a new memory mutation path, hosted dependency, MCP permission, auto-promote path, remote sync behavior, or background automation trigger.

The goal is to reduce setup surprise before stable v0.7.0.

# Vault-for-LLM v0.7.16

v0.7.16 is a hardening and productization release for agent-facing memory.

This release keeps the core local-first model, but makes the public surface
safer and more useful for multi-agent work:

- Local MCP reads default to `max_sensitivity=medium`.
- The GUI console now requires a token by default.
- `vault security doctor` checks GUI, MCP, and agent HMAC posture.
- Skill Registry now tracks versions, diffs, upgrade plans, and read-only MCP tools.
- Task Ledger records priority and due dates for active work.

Recommended upgrade:

```bash
pip install --upgrade "vault-for-llm[mcp]==0.7.16"
```

No manual migration is required. Vault applies the normal schema migration when
it opens a project database.

# Supabase UPSERT 重複

Supabase L2 UPSERT 出現重複插入問題，因 content_hash 包含時間戳導致每次不同，且舊 title 記錄與新記錄衝突。

## 關鍵要點
- content_hash 使用含時間戳的內容計算，導致每次執行 hash 不同
- 遷移腳本遺留舊 title 記錄與新記錄衝突
- 修復方案包括改用 stable_hash、刪除重複記錄與清理舊資料
- 最終從 24 筆減少至 3 筆

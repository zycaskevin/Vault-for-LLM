# Vault-for-LLM v0.6.69

This release connects transcript discovery to the automation inbox.

## What's New

- Added `vault automation inbox --include-transcripts`.
- Added `include_transcripts` and `transcript_limit` to MCP
  `vault_automation_inbox`.
- Inbox handoffs can now include `transcript_discovery` and
  `summary.uncaptured_transcripts`.
- Transcript hints remain metadata-only: filename/path, source-system hint,
  format, size, modified time, and `capture_path`.
- Transcript discovery is opt-in for inbox, so normal review remains fast and
  compact.

## Why It Matters

The closed loop now has a better next-agent handoff:

```text
scheduled automation -> inbox-latest.json -> candidate queue + uncaptured transcripts
```

The next agent can see both:

- which candidate memories need review, and
- which session exports may still need capture.

The safety boundary stays the same: discovery does not read transcript contents,
capture is preview-first, and active memory still requires explicit promotion.

## Try It

```bash
pip install "vault-for-llm[mcp]==0.6.69"
vault automation inbox \
  --project-dir ~/Vaults/my-project \
  --include-transcripts \
  --write-handoff \
  --pretty
```

For MCP, call `vault_automation_inbox` with:

```json
{
  "include_transcripts": true,
  "transcript_limit": 5,
  "write_handoff": true
}
```

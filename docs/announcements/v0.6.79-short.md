# Vault-for-LLM v0.6.79

This release lets MCP-capable agents start from the same local status and
handoff flow as the CLI.

## What Changed

- Added MCP `vault_update_status`.
- Added MCP `vault_automation_handoff`.
- Added both tools to the `core` MCP profile.
- Documented the read-first startup boundary in a decision record.

## Why It Matters

When several local agents share a machine, they can now ask Vault directly:

1. which version is installed,
2. which agents and vaults are registered,
3. which shared/private vault paths exist,
4. which compact handoff should be read before starting work.

The heavier review, capture, Dream, and maintenance tools stay outside the daily
startup path.

## Install

```bash
pip install "vault-for-llm[mcp]==0.6.79"
vault-mcp --project-dir ~/Vaults/project-memory --tool-profile core
```

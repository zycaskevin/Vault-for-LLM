# v0.6.97 Short Announcement

Vault-for-LLM v0.6.97 makes the automation loop a little smarter without
making it more dangerous.

`vault automation inbox` now reads the bounded learning policy created by
`vault automation eval --write-learning-policy`. If reviewers have repeatedly
promoted one kind of candidate, that group can move slightly earlier in the
review queue. If reviewers repeatedly reject another group, that group can be
slightly downgraded or kept under review.

The multiplier is visible, capped, and audit-friendly. It only changes review
ordering. It does not promote memory, delete memory, bypass privacy gates, or
change access policy.

# Vault-for-LLM v0.6.83

This release makes multi-Agent startup checks more personal without adding more
MCP tools.

## What Changed

- `vault update-status --agent <id>` adds a focused startup checklist.
- `vault update-status --read-status --agent <id>` reads the shared local status
  file and answers: does this Agent need attention?
- MCP `vault_update_status` now accepts `agent_id`.
- `setup-agent` writes startup contracts that include the configured Agent id.

## Why It Matters

One computer can have Codex, Claude Code, OpenClaw, Hermes, n8n, and other
runtime environments. They should share one machine-level update notice, but
each Agent should immediately know whether it personally needs an upgrade,
restart, or registry refresh.

This release keeps the MCP surface small: the existing `vault_update_status`
tool now returns `current_agent_notice`, `current_agent_needs_attention`, and
`startup_checklist` when `agent_id` is provided.

## Install

```bash
pip install "vault-for-llm[mcp]==0.6.83"
vault setup-agent
vault update-status --read-status --agent codex
```

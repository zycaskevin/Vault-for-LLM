# Vault-for-LLM v0.6.62

This release adds a full remote-reader doctor for Supabase-backed agents.

## What's New

- Added `vault remote doctor` to verify the whole remote reader path in one
  read-only check.
- The doctor checks search, UUID/integer IDs, guarded readable-entry RPCs,
  Document Map nodes, claims, content access, map, and bounded read.
- Failure responses include `failure_mode` and `next_action`, so an agent can
  explain what needs to be fixed without dumping private content.

## Why It Matters

Remote search can work while map/read still fails because the Supabase SQL,
Document Map sync, claims sync, or content access is incomplete. This command
turns that into a clear health report before Coze, n8n, Codex, Claude Code,
OpenClaw, Hermes, or another runtime depends on the remote vault.

## Upgrade Notes

If you use Supabase remote readers, upgrade and run:

```bash
pip install "vault-for-llm[supabase]==0.6.62"
vault remote doctor --agent-id remote-agent --query "deployment SOP" --json
```

Remote readers should use an anon or scoped authenticated key. Do not give
hosted agents a Supabase service role key.

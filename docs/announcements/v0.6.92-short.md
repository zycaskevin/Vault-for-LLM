# Vault-for-LLM v0.6.92

v0.6.92 adds a short, read-only activity feed for the memory automation closed
loop.

New command:

```bash
vault automation activity --pretty
```

New MCP tool:

```text
vault_automation_activity
```

Use it when an agent starts a session and needs to know what the automation loop
recently did: which low-risk candidates were previewed, which were promoted,
and which were skipped by policy or gates.

The boundary stays conservative. Activity is observability only: it does not
promote, reject, archive, delete, or expose raw candidate content.

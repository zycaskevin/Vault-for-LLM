# Vault-for-LLM v0.7.2

v0.7.2 is a small diagnostics and review-documentation patch for the stable v0.7 line.

It improves the sqlite-vec runtime story:

- `vault doctor` now distinguishes "sqlite-vec is installed" from "this SQLite runtime blocked extension loading".
- `vault stats` now reports legacy sqlite-vec embeddings and JSON-backed `semantic_vectors` separately.
- Runtime errors such as `sqlite3.OperationalError: not authorized` are shown as explicit degradations, not silent embedding loss.

This release also adds a corrected public v0.7.1 review note. The important wording is:

- `vault add` writes trusted active knowledge.
- `vault remember` creates candidate memory for agent/automation workflows.
- v0.7.1 was a CLI smoke hardening patch; the broader v0.7 line is the Agent Knowledge Platform foundation.

No new memory mutation path, hosted dependency, MCP permission, auto-promote path, remote sync behavior, or background automation trigger is added.

# Vault-for-LLM v0.6.26

This release closes the loop on agent-driven installation.

`vault setup-agent` can now do more than ask which optional features should be
enabled. When the user confirms, it can also install the selected optional
Python dependencies:

```bash
vault setup-agent \
  --non-interactive \
  --agent codex \
  --scope shared \
  --agent-project-dir ~/Vaults/my-project \
  --features core,mcp,semantic,supabase,headroom \
  --install-optional-deps \
  --install-embedding-model mix \
  --json
```

Highlights:

- Added `--install-optional-deps` for agent installers.
- Added `--install-embedding-model zh|en|mix` for semantic local model setup.
- Updated README variants, CLI reference, agent runbook, and manifest.
- Clarified that `/tmp/...` paths are disposable test workspaces, not stable
  shared Vault memory locations.

This makes Hermes/Nancy, Codex, OpenClaw, Claude Code, OpenCode, n8n, and other
agents less likely to stop at "next steps" when the user actually asked them to
install the selected Vault capabilities.

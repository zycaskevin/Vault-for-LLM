# Vault-for-LLM v0.6.80

This release makes the Agent installer generate MCP startup instructions, not
only CLI startup instructions.

## What Changed

- `vault setup-agent` now writes `agent-install/mcp-startup.json`.
- `vault setup-agent` now writes `agent-install/README-mcp-startup.md`.
- The generated startup sequence tells MCP-capable agents to call:
  1. `vault_update_status`
  2. `vault_automation_handoff`
  3. `vault_search`
  4. `vault_read_range`
  5. `vault_memory_propose`
- `local-smoke.sh` now verifies that the MCP `core` profile exposes the startup
  tools.

## Why It Matters

Different local runtimes can now install Vault once and share the same startup
contract. CLI-only agents can still use `vault update-status` and
`vault automation handoff`; MCP-capable agents can follow the generated MCP
startup guide directly.

## Install

```bash
pip install "vault-for-llm[mcp]==0.6.80"
vault setup-agent
```

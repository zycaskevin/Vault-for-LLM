## Vault-for-LLM v0.7.11

v0.7.11 hardens memory recall boundaries after the v0.7.10 review.

- Graph-expanded search now applies the same read policy as primary search
  results, so private or restricted neighbors cannot leak through graph hops.
- Search results now include `temporal_state` for fact-window metadata.
- `vault search --exclude-expired` and MCP
  `include_expired_temporal=false` let agents ask for currently valid facts
  while keeping past facts available for audit.
- Base64 payloads that decode to fail-level secrets now fail the privacy gate
  instead of only warning.

This release keeps temporal recall backwards-compatible: historical facts remain
searchable by default, but they are visibly marked.

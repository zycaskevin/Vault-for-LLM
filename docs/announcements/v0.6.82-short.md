# Vault-for-LLM v0.6.82

This release tightens the multi-Agent update flow.

## What Changed

- `vault update-status --read-status` reads the existing local update notice.
- MCP `vault_update_status` now supports `read_status=true`.
- `vault setup-agent` writes an update-status install pack:
  - `README-update-status.md`
  - `update-status-contract.json`
  - `update-status.cron`
  - `update-status.launchagent.plist`
- MCP startup guides now read existing status first, then fall back to local
  registry status without PyPI.

## Why It Matters

If Codex, Claude Code, OpenClaw, Hermes, n8n, or another local Agent all use the
same machine, one runtime can write the update notice and the others can read it.
The notice is metadata only: versions, registered Agents, vault paths, and
handoff commands. It is not an auto-upgrader.

## Install

```bash
pip install "vault-for-llm[mcp]==0.6.82"
vault setup-agent
vault update-status --read-status
```

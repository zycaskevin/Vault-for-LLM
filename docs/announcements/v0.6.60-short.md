# Vault-for-LLM v0.6.60

This release makes memory automation easier to teach.

## What's New

- `vault candidate-review` records rejected or blocked candidate decisions as audit-friendly feedback.
- MCP adds `vault_memory_review`, so reviewer agents can reject or block candidates without promoting them.
- Dream now proposes `consolidation_suggestion` candidates for duplicate memory groups instead of trying to merge or archive automatically.

## Why It Matters

Promotion is not the only useful signal. When a reviewer says "do not keep this" or "pause this until owner approval," Vault can now preserve that decision as structured feedback for future automation.

Dream also becomes safer around duplicates: it can point out likely consolidation work, but the merge/archive decision stays explicit and reviewable.

## Safety Boundary

This release does not add auto-promotion, hard delete, or permission bypass. The new paths write feedback events or review candidates only.

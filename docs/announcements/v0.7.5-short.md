# Vault-for-LLM v0.7.5

v0.7.5 turns the v0.7.3 memory features into a scheduled agent habit.

What changed:

- Generated cron, LaunchAgent, and n8n memory automation templates now run:
  1. `vault memory pipeline --write-candidates`
  2. `vault memory reflection --write-candidates`
  3. `vault automation cycle`
  4. `vault automation inbox --write-handoff`
  5. `vault automation learning-health --write-health`
- MCP adds review/maintenance tools for:
  - `vault_memory_pipeline`
  - `vault_memory_temporal_status`
  - `vault_memory_reflection`
- The `core` MCP profile stays unchanged to keep daily startup token cost small.

Safety boundary:

- Pipeline and reflection are candidate-first.
- No hard deletion is added.
- No hidden active-memory promotion is added.
- Generated handoffs remain compact and avoid raw transcript content by default.

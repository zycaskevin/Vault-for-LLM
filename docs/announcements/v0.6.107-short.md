# Vault-for-LLM v0.6.107

This release adds a doctor for generated Agent startup contracts.

```bash
vault agent startup-doctor --template-dir /path/to/project/agent-install --json
```

It checks whether a setup pack is current enough for the fleet-aware startup
flow:

1. `vault_update_status`
2. `vault_automation_handoff`
3. read `fleet_health_content` before `content`
4. search/read only when needed
5. propose durable memory as candidates

The doctor verifies:

- `mcp-startup.json`
- `adapter-startup-contract.json`
- `runtime-update-playbook.json`
- Codex, Claude Code, OpenClaw, and Hermes startup templates
- generated startup README files

Safety boundary: this is read-only. It does not edit runtime files, read private
memory, promote memory, or run live update checks.

# Vault-for-LLM v0.7.0

v0.7.0 is the first stable Agent Knowledge Platform foundation release.

This release promotes the validated v0.7 release-candidate line to stable. The focus is not a new hosted dependency or a new automatic memory mutation path. The focus is a coherent product surface:

- local-first Markdown + SQLite memory
- Agent-driven installation through `vault setup-agent`
- MCP startup, search, bounded read, memory proposal, automation brief, and handoff tools
- hybrid shared/private memory layouts
- multi-runtime startup templates for Codex, Claude Code, OpenClaw, and Hermes Agent
- optional Supabase remote-reader/sync templates
- report-first automation and candidate-first memory writes
- release gates for package parity, privacy scan, module size, README command smoke, Search QA, and PyPI clean install

The v0.7.0 line was validated through clean PyPI installs, `setup-agent`, `startup-doctor`, README minimal-path smoke, MCP memory/automation smoke, and CI across Python 3.10, 3.11, and 3.12.

Smoke-test clarification: active search/read checks should use a temporary `vault add` note; `vault remember` is intentionally candidate-first and does not appear in normal search until reviewed and promoted.

# Vault-for-LLM v0.6.55

This release adds the first feedback loop for memory automation.

Vault can already propose Dream and Forgetting candidates without promoting or
deleting memory automatically. v0.6.55 adds the missing next step: when those
candidates are promoted, rejected, or blocked, Vault records the outcome as
auditable feedback. Agents can then ask which kinds of suggestions are actually
earning trust over time.

Highlights:

- New schema v10 table: `memory_feedback_events`.
- Candidate outcomes now record promoted, rejected, and blocked events.
- New `vault automation eval` command summarizes feedback by source, memory
  type, category, acceptance rate, and recommendation.
- Feedback guides future curation priority only. It does not auto-promote,
  hard-delete, or override privacy/access policy.

Smoke test:

```bash
pip install "vault-for-llm[mcp]==0.6.55"
vault remember "Feedback smoke" \
  --content "Feedback smoke records candidate outcome after promotion." \
  --reason "Verify automation feedback loop" \
  --tags automation,feedback \
  --mode candidate
vault automation eval --pretty
```

Expected behavior: `automation eval` reports candidate feedback readiness and
source/type scores when promoted, rejected, or blocked outcomes exist.

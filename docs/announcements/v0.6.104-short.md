# Vault-for-LLM v0.6.104

This release adds a shared automation health panel for multi-Agent installs.

`vault automation fleet-health` gives humans, dashboards, and connected agents a
single read-only view across:

- local Agent registry metadata,
- automation learning-health status,
- update-distribution health.

```bash
vault automation fleet-health --write-health
```

That creates:

- `reports/automation/fleet-health-latest.json`
- `reports/automation/fleet-health-latest.md`

It is intentionally conservative: it does not read private memory, raw candidate
content, raw feedback reasons, or grant any new promotion authority.

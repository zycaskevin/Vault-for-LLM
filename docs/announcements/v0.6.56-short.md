# Vault-for-LLM v0.6.56

This release adds the next step in automation learning.

Vault already records whether memory candidates were promoted, rejected, or
blocked. v0.6.56 turns that feedback into a bounded learning policy that future
Dream, curator, or scheduled maintenance agents can read.

## Highlights

- `vault automation eval` now returns a `learning_policy` object.
- `vault automation eval --write-learning-policy` writes
  `reports/automation/learning_policy.json`.
- Learning rules can suggest `prefer_candidates`,
  `downgrade_or_require_review`, `keep_observing`, or `observe`.
- Priority multipliers are intentionally small, capped between `0.85` and
  `1.15`.

The policy is guidance only. It does not auto-promote candidates, delete
memory, or bypass privacy and access rules.

## Install

```bash
pip install "vault-for-llm[mcp]==0.6.56"
```

## Try It

```bash
vault automation eval --write-learning-policy --pretty
```
